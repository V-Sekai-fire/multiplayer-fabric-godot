# Contributing to multiplayer-fabric-godot

This is the V-Sekai-fire fork of Godot Engine. All custom work lives on
feature branches; the `multiplayer-fabric` branch is generated by
`multiplayer-fabric-merge` and must never be committed to directly.

## Branch rules

| Branch | Purpose |
|--------|---------|
| `feat/module-keychain` | `modules/keychain` + `thirdparty/keychain` |
| `feat/module-lasso` | `modules/lasso` (includes E2BIG fix) |
| `feat/engine-patches-NNN` | Engine patches: core, scene, servers, editor, drivers, tests |
| `feat/engine-patch-control-gui-input` | Expose `Control.call_gui_input` with viewport isolation |
| `archived/module-http3` | `modules/http3` + `thirdparty/picoquic` + mbedtls |
| `feat/module-multiplayer-fabric-NNN` | `modules/multiplayer_fabric` |
| `feat/module-multiplayer-fabric-mmog-NNN` | `modules/multiplayer_fabric_mmog` + predictive BVH + Lean proofs |
| `feat/module-open-telemetry` | `modules/open_telemetry` |
| `feat/module-sandbox` | `modules/sandbox` (godot-sandbox RISC-V integration) |
| `feat/module-speech` | `modules/speech` |
| `feat/module-sqlite` | `modules/sqlite` |
| `docs/contributing` | This file |

Branches suffixed `-NNN` are versioned snapshots. When a branch cannot be
force-pushed (protected), increment the number and update `gitassembly` in
`multiplayer-fabric-merge`.

## Making changes

Work on the appropriate feature branch, not on `multiplayer-fabric`:

```sh
git checkout feat/module-multiplayer-fabric-001
# make changes
git add <files>
git commit -m "Describe the change"
git push origin feat/module-multiplayer-fabric-002
# update gitassembly in multiplayer-fabric-merge to reference the new branch
```

If the branch is protected on GitHub, create a new versioned branch and
update `gitassembly` rather than force-pushing.

## Assembly

The `multiplayer-fabric` branch is assembled by running in
`multiplayer-fabric-merge`:

```sh
elixir update_godot_v_sekai.exs
```

This fetches all feature branches from this repo via the `v-sekai-fire`
remote and merges them in the order defined in `gitassembly`.

## Commit style

Sentence case. No `type(scope):` prefixes.

```
# good
Add RelZone::aoi_band_cells to replace _hilbert_aoi_band

# bad
feat(zone): add aoi band cells
```

## sandbox ↔ godot-sandbox integration

`multiplayer-fabric-sandbox` produces RISC-V ELF kernels consumed by the
`godot-sandbox` module (`modules/sandbox`). Changing a public kernel symbol
requires updating the binding table in `modules/sandbox` in the same commit.
Build and test by running the godot fork headless after replacing the ELF.

## Testing

```sh
godot --headless --quit   # smoke test; exits 0 on success
```
