# AGENTS.md

Guidance for AI coding agents working in this repository.

## What this repo is

The V-Sekai-fire fork of Godot Engine. Custom modules and engine patches
live on feature branches. The `multiplayer-fabric` branch is generated by
`multiplayer-fabric-merge` and is overwritten on every assembly run â€”
never commit unique work to it directly.

Branches suffixed `-NNN` are versioned snapshots created when a branch is
protected and cannot be force-pushed. Increment the number and update
`gitassembly` in `multiplayer-fabric-merge`.

## Rules that apply everywhere

- **Commit messages use sentence case.** First word capitalised, rest
  lowercase unless a proper noun. No `type(scope):` conventional-commit
  prefixes.
- **Never commit to `multiplayer-fabric`.** It is overwritten by assembly.
  All real work goes on a feature branch.
- **Versioned branches over force-push.** If a branch is protected, create
  `feat/foo-002` and update `gitassembly` â€” do not bypass protection.

## Assembly conflict rules

`.gitignore` uses the `union` merge driver (set in `.git/info/attributes`
of the `multiplayer-fabric-merge` working tree). Any file that causes an
add/add conflict across two feature branches must be moved to whichever
branch is merged first, removed from the later branch, and both branches
versioned.

## sandbox â†” modules/sandbox integration

`multiplayer-fabric-sandbox` produces RISC-V ELF kernels consumed by
`modules/sandbox`. Changing a public kernel symbol requires updating the
binding table in `modules/sandbox` in the same commit. Test by building
the fork headless with the updated ELF.
